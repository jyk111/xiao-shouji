# 音乐模块当前整合要求

## 边界

- 只改音乐模块相关内容。
- 不做网易云 / QQ 音乐登录界面；如果不能真实登录和播放，就不要放假入口。
- 不在界面里教育用户版权，不放“正版/盗版”说教文案。
- B 站、网易云、QQ 音乐不作为抓音频来源。

## 曲库

- 顶部入口只有：`曲库 / 听歌 / char创作`。
- 曲库负责找歌、导入歌、整理歌。
- 曲库内入口是一行一行：`我喜欢`、`歌单`、`历史`。
- 搜索结果默认没有内容，用户搜索后才出现。
- 不再把测试音源混进搜索结果。
- 搜索来源先接开放音乐源 Audius，让结果尽量是真能播放。
- 搜索结果标识只写播放状态：`可听`、`试听`、`不可播`。
- 搜索和上传分开：
  - 搜索音乐
  - 我的音乐：音频 URL、上传 MP3、测试声音
- 点击歌曲可自动加入曲库、自动加标签，也可加入我喜欢或歌单。

## 听歌

- 第二个界面就是听歌页，不做四个方块。
- 视觉参考网易云听歌页：背景图、唱片轮盘、歌词切换、底部控制。
- 按键必须高对比，不能白糊在灰背景里。
- 支持暂停/继续、上一首/下一首、随机、循环、队列。
- 支持选择 char 一起听，结束后记录 char 和听了多久。
- 尽量保持全局音频，切到其他模块仍继续播。

## char 创作

- char 创作要像真的创作歌曲，不只是写歌词。
- 发送给 AI 的提示词要参考：
  - char 人设
  - 世界书/浏览器世界书
  - 过去这个 char 创作过的歌曲
  - 当前填写的歌名、情绪、歌词方向
- 产出应包含：
  - 歌名
  - 风格/情绪
  - 旋律描述
  - 编曲建议
  - 歌词
- 创作好的歌曲保存为 `char 创作` 歌曲，归到对应 char 名下。
- char 创作歌曲可以收藏。
- 有 TTS 才允许试唱；没 TTS 就只提示先接 TTS。
