# 小剧场模块文档

## 这版文档的目的

当前只整理需求，不修改代码。

这份文档用于明确小剧场下一版应该怎么做，尤其是：

- 风格怎么处理
- 长度怎么设置
- 世界书 / low 随机怎么工作
- 主题库怎么导入、保存、选择
- 收藏、保存、删除应该是什么界面逻辑
- 哪些现有功能意义不清，需要改掉或重新定义

## 核心定位

小剧场不是普通聊天，也不是日记工具。

它应该是一个剧情生成器：

1. 玩家选择角色。
2. 玩家选择主题，或从主题库里选一个主题。
3. 玩家可以使用默认 low 世界书随机，或导入自己的世界书/主题库。
4. 系统像酒馆一样处理 `{{random:...}}`。
5. 生成一段小剧场。
6. 小剧场可以保存、收藏、删除、再次打开。

## 风格

### 当前问题

现在只有这些风格：

- 日常
- 暧昧
- 吵架
- 梦境
- 悬疑

这些太少，也太常见。更重要的是，如果玩家已经使用世界书随机，那么风格就不应该强行影响生成。

### 新设计

风格区域应该有两类：

### 固定风格

保留基础风格，例如：

- 日常
- 暧昧
- 吵架
- 梦境
- 悬疑

后续可以继续扩充。

### 随机风格

新增一个“随机”选项。

含义：

- 如果选择“随机”，就不再使用固定风格。
- 系统直接看 low 世界书 / 世界书随机结果。
- 生成提示词里不再额外写“风格：日常/暧昧/吵架”等。

换句话说：

```txt
选了固定风格：角色 + 主题 + 固定风格 + 可选世界书
选了随机风格：角色 + 主题/主题库 + low世界书/世界书随机
```

### 不要使用“无风格”

不要做一个叫“无”的风格选项。

如果玩家想完全交给世界书，那就选择“随机”。随机的意思就是：风格由世界书内容决定。

## 长度

长度需要从现在的“短/中/长”变成明确字数。

### 建议字数

- 短：约 200 到 600 字
- 中：约 400 到 800 字
- 长：约 800 到 1500 字

### 自由设置

在短、中、长之外，新增“自由设置”。

自由设置允许玩家输入：

```txt
约 1200 字
约 3000 字
约 500 字
```

生成时提示词应该写成：

```txt
请生成约 1200 字左右的小剧场。
```

而不是只靠模型 token 猜长度。

## 主题库

### 当前问题

现在主题主要靠玩家手动输入。

这不够。玩家应该可以导入主题库，并保存主题，然后从不同主题里选择。

### 新设计

小剧场应该有“主题库”。

主题库可以：

- 手动新增主题
- 导入主题 JSON / 文本
- 保存主题
- 删除主题
- 收藏主题
- 按分类选择主题
- 从主题库随机抽取主题

### 主题库里的主题可以是什么

主题可以是普通文本：

```txt
雨夜误会
旧友重逢
偷看手机后冷战
```

也可以是酒馆 random 格式：

```txt
{{random:雨夜误会,旧友重逢,秘密同居,梦中重逢}}
```

### 圈 / 分类

玩家提到“可以选择其他的圈”。

这里建议做成主题分类，比如：

- 默认
- 日常
- 狗血
- 梦境
- 校园
- 古代
- 玩家导入 A
- 玩家导入 B

玩家导入主题后，可以保存到某个分类里。之后不用每次手动输入，可以直接从分类里选。

## 世界书 / low 随机

### 当前问题

现在世界书 Roll 的表现很迷惑：

- 点随机后，界面突然出现很多内容。
- 它像是把所有 random 块都展开了。
- 玩家不知道它到底抽了什么。
- 它还会影响主题框，导致主题变得很长。

这不对。

### 正确目标

需要做成酒馆式 random。

酒馆逻辑是：

```txt
{{random:选项A,选项B,选项C}}
```

每次遇到这一段，只从里面抽一个。

比如：

```txt
地点：{{random:校园,医院,出租屋}}
关系：{{random:旧友,敌人,临时恋人}}
```

一次 Roll 后应该得到：

```txt
地点：医院
关系：旧友
```

不是把校园、医院、出租屋全部列出来。

### low 世界书

这里把“low 世界书”理解为：小剧场内置的一套默认随机库。

它应该默认存在，不需要玩家每次导入。

作用：

- 给“随机风格”提供随机来源。
- 给没有主题时提供剧情设定。
- 给玩家快速 Roll 一个小剧场方向。

### low 世界书应该怎么用

如果玩家选择“随机风格”，系统就看 low 世界书。

流程：

1. 从启用的 low 世界书里取内容。
2. 按酒馆 random 规则替换。
3. 得到“本次 low 结果”。
4. 本次 low 结果只显示在“随机结果预览”里。
5. 生成时把它发给模型。
6. 不要自动塞进主题框。

### 玩家导入世界书

玩家也可以导入自己的世界书。

导入后应该保存下来，之后一直存在，除非玩家删除。

世界书条目应该支持：

- 启用 / 停用
- 是否参与随机
- 查看内容
- 删除
- 重新导入覆盖
- 分类或来源名称

### 随机按钮应该做什么

随机按钮只应该做一件事：

```txt
生成本次随机结果
```

它不应该：

- 自动打开所有世界书条目
- 自动把所有条目塞到主题框
- 自动把界面撑得很长
- 让玩家不知道发生了什么

随机后只显示一个清晰结果：

```txt
本次随机结果
地点：医院
关系：旧友
冲突：误会升级
```

然后提供按钮：

- 使用这个结果生成
- 重新随机
- 保存为主题
- 复制到主题

其中“复制到主题”应该是明确动作，不应该默认发生。

## 生成逻辑

### 固定风格生成

如果玩家选择固定风格：

```txt
角色 + 主题 + 固定风格 + 长度 + 可选最近聊天 + 可选世界书结果
```

### 随机风格生成

如果玩家选择随机：

```txt
角色 + 主题/主题库 + low世界书随机结果 + 长度 + 可选最近聊天
```

此时不要再额外添加：

```txt
风格：日常/暧昧/吵架/梦境/悬疑
```

### 主题和随机结果的关系

主题和随机结果应该分开。

主题是玩家明确选择或输入的核心方向。

随机结果是本次生成参考。

二者可以同时存在，但随机结果不要自动污染主题框。

## 保存后的作品界面

### 当前问题

生成后虽然可以保存，但保存后的界面不够明确。收藏和删除应该在作品详情里出现，而不是让玩家猜“收藏点了会不会打开新界面”。

### 新设计

生成完成后进入“作品详情 / 生成结果”区域。

作品详情里应该有：

- 标题
- 正文
- 使用的角色
- 使用的主题
- 使用的随机结果
- 使用的长度
- 创建时间

底部操作：

- 保存
- 收藏 / 取消收藏
- 删除
- 重新生成
- 回到编辑

### 删除需要二次确认

删除不能直接删。

应该弹出确认：

```txt
确定删除这个小剧场吗？
删除后不可恢复。

取消 / 确定删除
```

## 收藏

### 当前问题

现在点击“收藏”不会打开新界面，所以玩家会觉得“点不进去”。

### 新定义

收藏不是页面入口，而是一个作品状态。

但界面上应该有“收藏夹 / 已收藏”视图，让玩家能看到收藏了哪些小剧场。

### 建议

历史列表增加筛选：

- 全部
- 收藏

点击“收藏”按钮只改变当前作品状态。

点击“收藏筛选”才进入收藏列表。

这样就不会误解为“收藏按钮应该打开新界面”。

## 存日记

### 当前问题

“存日记”意义不清。

它现在像是把小剧场复制到日记，但小剧场本身已经有保存和历史，所以这个功能容易重复。

### 建议

先不要把“存日记”放在一级按钮。

可以放进更多菜单，或者改名：

```txt
转成日记
```

并明确说明：

```txt
把这段小剧场作为一篇日记保存到日记模块。
```

如果暂时没有清晰用途，可以先隐藏。

## 最近聊天

### 当前问题

最近聊天现在是隐形上下文。

玩家不知道：

- 有没有启用
- 取了哪些聊天
- 会不会影响剧情

### 建议

新增“最近聊天”开关。

状态：

- 关闭：不读取聊天上下文
- 开启：读取所选角色最近聊天

并提供“查看摘要”：

```txt
本次将参考：
- 角色A 最近 8 条微信
- 角色B 最近 6 条 QQ
```

## 页面结构建议

### 1. 创作设置

- 角色
- 主题来源：手写 / 主题库 / 随机主题
- 风格：日常、暧昧、吵架、梦境、悬疑、随机
- 长度：短、中、长、自由设置
- 最近聊天：开关

### 2. 主题库

- 导入主题
- 新建主题
- 分类 / 圈
- 随机抽主题
- 保存主题

### 3. 世界书 / low

- 默认 low 世界书
- 玩家导入世界书
- 条目开关
- 参与随机开关
- 酒馆式 random 替换
- 本次随机结果预览

### 4. 生成结果

- 正文预览
- 保存
- 收藏
- 删除，带二次确认
- 重新生成

### 5. 历史

- 全部
- 收藏
- 按主题/分类筛选
- 打开详情

## 当前需要改的点清单

1. 风格增加“随机”。
2. 随机风格时，不再使用固定风格提示词，直接看 low 世界书/世界书随机。
3. 扩展长度：短 200-600 字，中 400-800 字，长 800-1500 字，自由设置约多少字。
4. 生成结果详情里放保存、收藏、删除。
5. 删除增加二次确认。
6. 收藏按钮只切状态，另做收藏筛选/收藏列表。
7. 隐藏或弱化“存日记”，避免一级功能意义不明。
8. 世界书随机改成酒馆式替换，不展开所有 random。
9. 随机结果单独显示，不自动塞进主题。
10. 内置默认 low 世界书。
11. 支持玩家导入世界书并持久保存。
12. 支持玩家导入主题库并持久保存。
13. 主题库支持分类/圈。
14. 最近聊天做成开关和可查看摘要。

## 一句话总结

小剧场应该变成“角色 + 主题库 + 默认 low 世界书/玩家世界书 + 酒馆式 random + 明确长度 + 可保存作品”的剧情生成器，而不是让随机结果把主题框塞满、让收藏按钮看起来像坏掉、让玩家不知道世界书到底抽了什么。
